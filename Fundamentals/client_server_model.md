# Client-Server Model

## Overview
- The client-server model describes how a client and server communicate with each other.

## Key Concepts
- Client - Device or program that requests a service or resource.
- Server - Device or program that provides the services or resources.
- Request - A message sent from a client asking for data or a service.
- Response - A server's reply to a request.
- Protocol - A set of rules to define how data is communicated across a network.
- Port - A communication endpoint used to identify specific services on a server.
- DNS - A human-readable address that translates to an IP address.
- IP address - A unique numerical label assigned to a device on a network.
- URL - Full web page address used to locate a resource on the web.

## HTTP & GET Requests
- HTTP/HTTPS - A stateless client-server protocol used for transferring web data, HTTPS is the secure version.
- GET Request - A HTTP method used by a client to request data from a server.

## Relevance
- Essential for understanding web communication, troubleshooting, and cybersecurity concepts.
