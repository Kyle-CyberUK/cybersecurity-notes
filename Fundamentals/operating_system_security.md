# Operating System Security

## Overview
- Operating systems are responsible for protecting users, data, and system resources.
- OS security ensures that only authorized users can access resources, sensitive information is safeguarded, and system stability is maintained.

## Authentication
- Verifies identity of users.
- Prevents unauthorized access to systems.

## Authorization and Access Control
- Determines what resources users are allowed to access.
- Protects sensitive files from normal users.

## Least Privilege
- Users should only be granted the minimum permissions necessary to perform their tasks.
- Reduces potential damage if an account is compromised.

## Secure Remote Access
Secure Shell (SSH) provides:
  - Encrypted communication.
  - Secure authentication.
  - Remote command execution.

## Privilege Escalation
- Refers to gaining higher-level access from a lower-privileged account.
- Usually exploited if security controls are weak.

## OS Protection Mechanisms
- Authentication systems to verify users.
- File permissions to control access.
- Process isolation to prevent interference between applications.
- Memory protection to prevent unauthorized memory access.
- Logging and auditing for monitoring and accountability.

## Relevance
- OS security is essential because the OS forms the boundary between users, applications, and hardware.
