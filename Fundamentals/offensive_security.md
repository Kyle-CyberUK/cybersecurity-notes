# Offensive Security

## Overview
- Offensive security involves ethically breaking into systems to identify weaknesses and vulnerabilities.

## Security Terms
- Red teaming - An authorized, simulated attack where security experts act as real attackers to test the effectiveness of security defences.
- Penetration testing - A controlled security test where an authorized tester exploits and identifies vulnerabilities.
- Vulnerability - A weakness in a system that could be used by an attacker to gain control.
- Exploit - A method or tool used to gain advantage of a vulnerability.
- Scope - The rules and regulations of what systems and actions are allowed during a test.

## Reconnaissance and Enumeration
- Attackers begin by searching for hidden pages and resources on a website.
- For example:
  - Manually testing common paths such as /admin and /login.
  - Automated directory discovery.
- Gobuster is a tool used to discover hidden directories on web servers using wordlists.

## Password Attacks
- Attackers may attempt to gain access by testing possible passwords.
- A dictionary attack tests a list of common passwords against a login system.
- Hydra is an automated tool, often used in Kali Linux, to perform brute-force and dictionary attacks.

## Relevance
- By identifying vulnerabilities, companies can strengthen their systems and reduce security risks.
